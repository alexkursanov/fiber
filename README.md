# Модель электро-механического сопряжения кардиомиоцита

Данный проект реализует математическую модель электро-механического сопряжения кардиомиоцита на основе кода MATLAB.

## Структура проекта

```
cardiac_model/
│
├── config/
│   ├── __init__.py
│   └── parameters.py           # Классы параметров
│
├── models/
│   ├── __init__.py
│   ├── electrical.py           # Электрофизиологическая модель (TNNPE)
│   ├── mechanical.py           # Механическая модель (l2l3, v_meh)
│   └── diffusion.py            # Модель диффузии (PDE)
│
├── core/
│   ├── __init__.py
│   ├── solver.py               # Основной решатель
│   ├── state.py                # Глобальное состояние
│   └── utils.py                 # Вспомогательные функции
│
├── visualization/
│   ├── __init__.py
│   ├── plots.py                 # Функции построения графиков
│   └── animation.py             # Анимации
│
├── io/
│   ├── __init__.py
│   ├── h5_io.py                  # Сохранение/загрузка в HDF5
│   └── import_data.py             # Импорт из Excel/MAT
│
├── tests/
│   ├── __init__.py
│   └── test_model.py              # Тесты
│
├── notebooks/
│   └── analysis.ipynb             # Jupyter для анализа
│
├── main.py                         # Основной скрипт запуска
├── requirements.txt                # Зависимости
└── README.md                       # Документация
```

## Установка

```bash
pip install -r requirements.txt
```

## Использование

### Базовый запуск
```bash
python main.py
```

### С параметрами
```bash
python main.py --ischemia 10 --duration 5000 --cells 100 --output my_results.h5
```

### Возобновление расчета
```bash
python main.py --checkpoint results_checkpoint.h5
```

### Без графиков
```bash
python main.py --no-plot
```

## Визуализация

После расчета создаются:
- Графики потенциала действия
- Кальциевые транзиенты
- Механические переменные
- Пространственно-временные диаграммы
- Анимации (при наличии ffmpeg)

## Формат данных

Результаты сохраняются в HDF5 формате:
- `results/V` - потенциал действия
- `results/Ca_i` - внутриклеточный кальций
- `results/Ca_SR` - кальций в SR
- `results/Na_i` - натрий
- `results/K_i` - калий
- `results/N` - вероятность прикрепления мостиков
- `results/v` - скорость сокращения
- `results/l1`, `l2`, `l3` - длины элементов
```

## Оптимизация производительности

1. **Numba JIT** - для критических вычислений
2. **Векторизация** - где возможно
3. **Профилирование** - для поиска узких мест
4. **Параллелизация** - клетки можно считать параллельно
5. **Контрольные точки** - для возобновления расчетов
